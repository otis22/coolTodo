## Описание

Написать E2E тест для проверки создания задачи через UI. Тест должен открыть фронтенд, ввести текст задачи, нажать Enter и проверить, что задача появилась в списке.

## Зависимости

- [x] D3.1: Настроить Laravel Dusk для E2E тестов (Completed ✅)
- [x] C8: Интегрировать фронтенд с бэкендом (Completed ✅)

## Текущее состояние

Базовый тест `ExampleTest` существует, но проверяет только загрузку страницы. Нужно создать полноценный тест для создания задачи.

## Требования

### Цель

Создать E2E тест, который:
- Открывает фронтенд приложение
- Вводит текст задачи в поле ввода
- Нажимает Enter для создания
- Проверяет, что задача появилась в списке
- Проверяет, что счетчик обновился

### Функциональность

1. **Открытие приложения**:
   - Переход на `http://localhost:5173`

2. **Создание задачи**:
   - Поиск поля ввода с placeholder "What needs to be done?"
   - Ввод текста задачи
   - Нажатие Enter

3. **Проверка результата**:
   - Проверка, что задача появилась в списке
   - Проверка текста задачи
   - Проверка, что счетчик обновился (если был пустой список, то "1 item left")
