# Task A15: Исправить версию Xdebug в Dockerfile.dev

**Статус**: In Progress  
**Приоритет**: Medium  
**Оценка**: 0.1 дня

## Описание

В `Dockerfile.dev` используется `pecl install xdebug` без указания версии, что может привести к установке версии 4.x или выше при обновлении PECL. Это нарушает требование задачи A9 о фиксации версии Xdebug 3.x.

## Проблема

**Файл**: `Dockerfile.dev`  
**Строка**: 19  
**Текущий код**:
```dockerfile
RUN pecl install xdebug && docker-php-ext-enable xdebug
```

**Ожидаемое поведение**: Версия Xdebug должна быть зафиксирована на 3.x, как в основном `Dockerfile` (используется `xdebug-3.3.3`).

## Причина

При выполнении задачи A9 была обновлена версия Xdebug только в основном `Dockerfile`, но не в `Dockerfile.dev`. В `Dockerfile.dev` осталась команда без версии, что может привести к установке несовместимой версии в будущем.

## Решение

Обновить строку 19 в `Dockerfile.dev`:
```dockerfile
RUN pecl install xdebug-3.4.7 && docker-php-ext-enable xdebug
```

**Примечание**: Версия 3.3.3 недоступна через PECL (проверено). Используется версия 3.4.7 - последняя доступная версия 3.x через PECL. Это соответствует требованию задачи A9 о фиксации версии 3.x.
