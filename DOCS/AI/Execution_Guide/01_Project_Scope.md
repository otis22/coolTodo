# Project Scope: CoolTodo

## Objective

Разработать веб-сервис для управления персональным списком задач (заметками, "todo"). Одностраничное веб-приложение (SPA), позволяющее пользователю создавать, просматривать, редактировать и удалять задачи. Все изменения немедленно сохраняются на сервере для синхронизации между устройствами.

## Deliverables

### Core Library (Backend)
- RESTful API для управления задачами
- Domain Layer с бизнес-логикой (Use Cases, Models)
- Infrastructure Layer (Repositories, Controllers)
- База данных MySQL с миграциями

### Web Application (Frontend)
- Vue 3.5 SPA с компонентами TodoList и TodoItem
- Интеграция с backend API через сервисный слой
- Фильтрация задач (All, Active, Completed)
- Реактивный интерфейс с немедленным сохранением

### DevOps & Quality
- Docker окружение для разработки и production
- CI/CD пайплайн (GitHub Actions)
- Полное покрытие тестами (Unit, Integration, E2E)
- Статический анализ кода (PHPStan level 9)

## Constraints

### Технические ограничения

**Версии технологий**:
- PHP 8.3 (строгая типизация обязательна)
- Laravel 11.x
- MySQL 8.0/8.4
- Vue 3.5
- Vite 7.2

**Качество кода**:
- PHPStan level 9 (максимальный уровень)
- Покрытие тестами Domain Layer >90%
- Все тесты должны проходить перед коммитом
- Код должен соответствовать PSR-12

**Производительность**:
- Выдерживать пиковую нагрузку 100 RPS
- Время ответа API < 200ms для 95% запросов
- Ограничение ресурсов: 2 vCPU, 8 ГБ RAM

**Процесс разработки**:
- Always Green Main Branch (каждый коммит проходит CI)
- Test-First подход (TDD)
- Минимальные PR (~150 строк)
- Один PR = одна задача (где возможно)

### Архитектурные ограничения

**Разделение слоев**:
- Domain Layer не должен зависеть от Laravel
- Infrastructure Layer реализует интерфейсы из Domain
- Четкое разделение бизнес-логики и инфраструктуры

**12-Factor App**:
- Все настройки через environment variables
- Stateless процессы
- Логи как потоки событий
- Конфигурация через `.env`

### Функциональные ограничения

**Версия 1.0**:
- Один пользователь (без аутентификации в первой версии)
- Локальное хранилище файлов
- Логирование в файлы
- Очереди через database driver

**Будущие версии** (out of scope для v1.0):
- Мультипользовательская система
- Облачное хранилище (S3)
- Интеграция с GlitchTip для логов
- Redis/SQS для очередей

## Success Criteria

Проект считается успешным, если:

1. ✅ Все функциональные требования из ТЗ реализованы
2. ✅ Все тесты проходят (Unit, Integration, E2E)
3. ✅ PHPStan level 9 без ошибок
4. ✅ Приложение работает в Docker
5. ✅ CI пайплайн проходит успешно
6. ✅ Производительность соответствует требованиям (100 RPS)
7. ✅ Документация актуальна и полна

## Out of Scope

Следующие функции **не входят** в scope версии 1.0:

- Аутентификация и авторизация пользователей
- Мультипользовательская система
- Облачное хранилище файлов
- Интеграция с внешними сервисами (кроме базовых)
- Мобильные приложения
- Офлайн-режим с синхронизацией
- Расширенные уведомления
- Экспорт/импорт данных

Эти функции могут быть добавлены в будущих версиях.
