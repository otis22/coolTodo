Установить и настроить Laravel Dusk для написания E2E (End-to-End) тестов. Создать базовую конфигурацию и проверить, что Dusk работает корректно.

## Текущее состояние

Laravel Dusk может быть установлен или нет. Нужно проверить и настроить:
- Установка Laravel Dusk через Composer
- Настройка конфигурации Dusk
- Создание базового теста для проверки работы
- Настройка окружения для запуска тестов

## Требования

### Цель

Настроить Laravel Dusk для E2E тестирования:
- Dusk установлен и настроен
- Базовый тест создан и проходит
- Окружение готово для написания E2E тестов

### Функциональность

1. **Установка**:
   - Установить Laravel Dusk через Composer
   - Настроить конфигурацию

2. **Базовый тест**:
   - Создать простой тест для проверки работы Dusk
   - Убедиться, что тест проходит

3. **Конфигурация**:
   - Настроить `.env.dusk.local` если нужно
   - Настроить ChromeDriver или использовать Docker

## План выполнения

### Шаг 1: Проверка установки
- [ ] Проверить, установлен ли Laravel Dusk
- [ ] Если нет - установить через `composer require --dev laravel/dusk`

### Шаг 2: Настройка Dusk
- [ ] Запустить `php artisan dusk:install` если нужно
- [ ] Проверить конфигурацию в `tests/DuskTestCase.php`
- [ ] Настроить базовый URL для тестов

### Шаг 3: Создание базового теста
- [ ] Создать простой тест (например, проверка загрузки главной страницы)
- [ ] Убедиться, что тест проходит

### Шаг 4: Настройка окружения
- [ ] Проверить работу ChromeDriver или настроить Docker
- [ ] Убедиться, что тесты могут запускаться
