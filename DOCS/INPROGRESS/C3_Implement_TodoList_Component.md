## Описание

Реализовать компонент `TodoList` для отображения списка задач. Компонент должен использовать `TodoItem` для отображения каждой задачи, загружать данные через API и обрабатывать события от `TodoItem`.

## Зависимости

- [x] C2: Реализовать компонент TodoItem (Completed ✅)

## Текущее состояние

В `frontend/src/components/TodoList.vue` есть только заглушка с TODO комментариями.

## Требования

### Цель

Создать компонент, который:
- Загружает список задач через API (`getTodos()`)
- Отображает список через компонент `TodoItem`
- Обрабатывает события от `TodoItem` (updated, deleted)
- Обновляет список после изменений

### Функциональность

1. **Загрузка данных**:
   - При монтировании компонента загружать задачи через `getTodos()`
   - Показывать состояние загрузки
   - Обрабатывать ошибки загрузки

2. **Отображение списка**:
   - Использовать `TodoItem` для каждой задачи
   - Передавать todo объект в props
   - Обрабатывать события `@updated` и `@deleted`

3. **Обновление списка**:
   - При обновлении задачи обновлять локальное состояние
   - При удалении задачи удалять из списка
   - Реактивно обновлять UI

### Примечание

Фильтрация (All/Active/Completed) будет реализована в задаче C4, поэтому в C3 её не нужно делать.

## Реализация

### Реализованные функции

1. **Состояние компонента**:
   - `todos` - реактивный список задач
   - `isLoading` - состояние загрузки
   - `error` - сообщение об ошибке

2. **Методы**:
   - `loadTodos()` - загружает задачи через API `getTodos()`
   - `handleTodoUpdated(updatedTodo)` - обновляет задачу в списке
   - `handleTodoDeleted(todoId)` - удаляет задачу из списка

3. **Lifecycle**:
   - `onMounted()` - загружает задачи при монтировании компонента

4. **Интеграция с TodoItem**:
   - Передает `todo` объект в props
   - Обрабатывает событие `@updated`
   - Обрабатывает событие `@deleted`
